Young 编辑器 (个人项目) 2023.11 ~ 至今
Young 编辑器是一个类似 Figma(行业标杆) 的可视化编辑软件, 由本人独立制作, 可编辑矢量图形、图片、文字等. 本编辑器致力于高性能编辑, 以下是其与 Figma 部分性能对比: https://v.douyin.com/iMj7hPya/
在线体验地址: https://gitborlando.github.io/... Github 地址: https://github.com/gitborlando/editor

本编辑器技术栈为 React + Vite, 比较核心的技术栈均为自研, 例如:

渲染引擎: 本编辑器的渲染库经过多次迭代, 初始是 Konva.js, 由于其性能不佳, 遂使用 Pixi.js, 编辑器性能得到大幅提升. 然而 Pixi.js 大多用于游戏开发, 因此编辑器的编辑能力受限制, 比如无法绘制环扇形, 文字段落无法应用多个样式等, 遂决定自行研发. 该自研引擎基于原生 Canvas, 维护一套类似 dom 树的渲染树, 能实现事件的捕获与冒泡传递. 使用了脏矩形检测、分片渲染、离屏渲染等多种绘制优化手段.

状态管理工具: Immui.js + Signal.js, 受 Immer 和 MobX 的启发, 由本人独自开发, 用于管理编辑器内的状态. Immui.js 是一个 Immutable 库, 自研目的在于更高效的性能优化(应用了一些 hack 手段). Signal.js 基于发布订阅模式, 其优势是更加细粒度的控制, 例如可自定义回调函数的触发顺序等.

碰撞检测: 编辑器点击和框选依赖于 图形学 碰撞检测, 本人参考 Pixi.js 部分源码重新设计了一套有向包围盒, 轴对齐包围盒, 任意曲线, 以及任意不规则图形的碰撞算法.
